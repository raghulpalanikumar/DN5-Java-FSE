# Week 1 – Design Patterns, Principles & Data Structures and Algorithms

## Overview

This week focused on building a strong foundation in software design and problem-solving. The topics covered include SOLID principles, commonly used design patterns, algorithm analysis, data structures, searching techniques, and sorting algorithms.

## Learning Objectives

* Understand and apply SOLID principles in object-oriented programming.
* Learn commonly used design patterns and their real-world applications.
* Analyze algorithm performance using time and space complexity.
* Implement and work with fundamental data structures.
* Apply searching and sorting techniques to solve programming problems efficiently.

## Topics Covered

### Design Principles

* Single Responsibility Principle (SRP)
* Open/Closed Principle (OCP)
* Liskov Substitution Principle (LSP)
* Interface Segregation Principle (ISP)
* Dependency Inversion Principle (DIP)

### Design Patterns

* Singleton Pattern
* Factory Method Pattern
* Builder Pattern
* Adapter Pattern
* Decorator Pattern
* Proxy Pattern
* Observer Pattern
* Strategy Pattern
* Command Pattern

### Data Structures and Algorithms

* Algorithm Analysis
* Time and Space Complexity
* Arrays
* Linked Lists
* Linear Search
* Binary Search
* Bubble Sort
* Insertion Sort
* Merge Sort
* Quick Sort

## Mandatory Hands-on Exercises

### 1. Implementing the Singleton Pattern

**Objective:** Create a class that allows only one instance throughout the application lifecycle.

### 2. Implementing the Factory Method Pattern

**Objective:** Create objects through a factory interface without exposing the object creation logic.

### 3. E-Commerce Platform Search Function

**Objective:** Implement efficient searching techniques for product retrieval and analysis.

### 4. Financial Forecasting

**Objective:** Apply algorithmic concepts and data structures to solve forecasting-related problems.

## Key Takeaways

* SOLID principles help in writing maintainable and scalable code.
* Design patterns provide reusable solutions to common software design problems.
* Understanding algorithm complexity is essential for optimizing application performance.
* Data structures and algorithms form the foundation of efficient software development.
